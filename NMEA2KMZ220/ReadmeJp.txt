
                               NMEA to KMZ Utility

                              NMEA2KMZ.exe Ver2.20

                               Jul. 2011 (C)4river
                     http://homepage2.nifty.com/k8/gps/indexj.htm

概要
  GPS受信機のNMEAログファイルをGoogle Earth用のKML(Keyhole Markup Language)に、
  またはKMLをZIP形式で圧縮したKMZファイルに変換します。
  また CSV, GPX, NMEA形式での出力も可能です。
  下記のGPSログファイルも変換できます。
    Wintec: WBT-200, WBT-201, WBT-202, WSG-1000
    RoyalTek: RBT-3000
    Transystem: i-Blueシリーズ
    Qstarz: BT-Q1000, BT-Q100X
    HOLUX: M-241
    Columbus: V-900
    VISIONTAC: VGPS-900
    Hanwha: Pocket GPS S1 (PG-S1)


  動作環境
    動作確認済OS: Windows 7, Window Vista, Windows XP.


特徴

  1)KMZ, KML, GPX, NMEA および CSVの５形式の出力を選択できる。
  2)軌跡ポイントに情報を出力できる（KMZ, KML）。
    データ番号、年月日、時刻、緯度、経度、速度、高度を選択して出力できる。
    速度の単位を Km/h, mph, kt に切替可能。
    時刻をローカルタイムに変換して出力できる。
  3)軌跡ポイントのアイコンを選択できる（速度に対応して４種類）（KMZ, KML）。
    外部アイコンファイルも利用できる。
  4)軌跡の線色、線幅、不透明度を指定できる（KMZ, KML）。
  5)複数のファイルを指定して一括変換が出来る。
  6)コマンドライン・オプションおよびドラッグ＆ドロップが有効。
    予め複数の設定ファイルを用意しておき、データファイル共にドラッグ＆ドロップできます。
    ショートカットを複数用意しておき、コマンドラインオプションに異なる条件の設定ファイルを
    指定しておくことが出来ます。
  7)３Ｄモードでの出力が可能で、出力条件を細かく設定できます（KMZ, KML）。
    速度を高さに置き換えて表示することも出来ます。
  8)出力範囲を時刻で限定できます。
  9)軌跡とプレースマークの出力をOn/Offできます（KMZ, KML）。
 10)プレースマークアイコンにユーザーが作成したアイコンを使用できます。
 11)タイトル用にユーザーが作成したロゴイメージを表示できます。
 12)GPS機種の判定条件と機能をユーザーが追加できます。
 13)ジオイド・グリッドデータを用いて緯度経度に応じてジオイド値を補正できます。
 14)インストールが不要なのでUSBメモリー上で実行できます。


インストール
  インストールは実行ファイル NME2KMZ.exe を適当なフォルダにコピーするだけです。
  レジストリは使用しないので NME2KMZ.exeと NME2KMZ.ini の削除でアンインストールできます。
  ジオイド補正を行う場合は別途ジオイドグリッドデータを入手して NME2KMZ.exe と同じフォルダに
  コピーする必要があります（設定ファイルの[Geoid]セクションのGeoidPathで別フォルダを指定できます）。
  CustomGeoidユーティリティーを使用してEGM2008から必要なエリアを切り出して使用することも出来ます。

  ジオイド・グリッドデータの入手先。
    EGM2008 2.5'x2.5': 
      National Geospatial-Intelligence Agency (NGA)
      NGA: EGM2008 - WGS 84 Version
      http://earth-info.nga.mil/GandG/wgs84/gravitymod/egm2008/egm08_wgs84.html
      2.5 x 2.5-Minute Geoid Undulation Grid in WGS 84 - SMALL ENDIAN
      Und_min2.5x2.5_egm2008_isw=82_WGS84_TideFree_SE.gz を解凍して
      Und_min2.5x2.5_egm2008_isw=82_WGS84_TideFree_SE (149,368,328byte) を得ます。
    EGM2008 1'x1':
      National Geospatial-Intelligence Agency (NGA)
      NGA: EGM2008 - WGS 84 Version
      http://earth-info.nga.mil/GandG/wgs84/gravitymod/egm2008/egm08_wgs84.html
      1 x 1-Minute Geoid Undulation Grid in WGS 84 - SMALL ENDIAN
      Und_min1x1_egm2008_isw=82_WGS84_TideFree_SE.gz  を解凍して
      Und_min1x1_egm2008_isw=82_WGS84_TideFree_SE (933,292,808) を得ます。
        注）EGM2008 2.5'x2.5'との差は約3.5mm以内なので通常は2.5'x2.5'で十分です。
    GSIGEO 2000:
      国土地理院のCD-ROMを入手するかWebサイトからダウンロードします。
      国土地理院 「日本のジオイド2000」ジオイド・モデル データ
      http://vldb.gsi.go.jp/sokuchi/geoid/download/down.html
      http://vldb.gsi.go.jp/sokuchi/geoid/download/agreement.html
      gsigeoid_ver5_asc.zip を解凍して programフォルダ下の gsigeome_ver5.asc (19,670,776byte) を使用します。


使用方法

1.入力ファイル名の指定
  Input file の File name の部分にソースファイルのファイル名を入力します。
  複数ファイルを指定する場合はセミコロン「;」で区切って入力します。

  右端のボタンをクリックするとオープン･ダイアログが開きます。
  複数のファイルを選択して一括変換が可能です。
  「ファイルの種類(T)」のプルダウンリストでファイルの種類を選択できます。

2.変換の実行
  「Convert」ボタンをクリックすると変換を実行します。
  変換が成功すると画面下部に「*** Completed: filename xxx records ***  xx files」と表示します。
  画面下部の「Report」をクリックすると詳細なレポートを表示します。
  「Esc」キーで実行を中断できます。

  出力ファイルは入力ファイルと同一のフォルダに作成されます。
  ファイル名はデータ開始時刻の年月日時分秒に張子名 .kml .kmz .gpx または .nmea を付加したものに
  なります。
    注）出力ファイルと同一ファイル名のファイルが存在する場合は上書きされます。

  Hanwha Pocket GPS S1 (PG-S1) のログファイルを入力した場合は、ログ開始時の緯度経度と基準高度を
  表示して、この地点の実際の標高と気温の入力を求めます。
  この場合は補正した標高と気圧をデータ出力できます。
    注1) ファームウェアのアップデートで使えなくなる可能性があります(10.04.23 で動作確認)。
    注2) 気圧と気温は時間と共に変動するため、長時間のログでは誤差が増大します。
    注3) 設定ファイルの[PG-S1]セクションの UseBarometer=1 を UseBarometer=0 に変更すると
         この機能を停止できます。
    注4) 設定ファイルの[PG-S1]セクションの BarometerOffset に気圧補正用オフセット値を設定できます。
    注5) 温度補償は温度プロファイル 6.5℃/Kmで簡易補正します(基準値: 1013.25hPa=0m at 15℃)。

3.オプション設定

  1)Use Fix only
    GPSが測位状態のデータのみを変換します(NMEA, i-Blueシリーズ, Qstarz BT-Q1000)。

  2)Use Estim.
    推測航法のデータも出力します。

  3)Split Time
    入力データが WBT-100,200,201,202,WSG-1000 以外の場合に出力ファイルを分割する欠測時間を指定します。
    0.25分（15秒）以下の設定は出来ません。

  4)Mask
    チェックするとHDOPが指定値以下で使用衛星数が指定値以上のデータのみを出力します。
    NMEA(GGAセンテンス付）、RBT-3000 および HDOP, NSAT付のi-BlueシリーズとBT-Q1000で有効です。
    チェック時には設定ダイアログが開きます。

  5)Reduce
    チェックすると指定時間以上の経過、または指定距離以上移動した場合のみデータを出力します。
    チェック時には設定ダイアログが開きます。

  6)Output in local time
    Point note の時刻をローカルタイムに変換して出力します。

  7)Output range limitting
    チェックすると出力データ範囲を時刻で指定できます。

    データファイルの読込完了時に出力範囲設定用の「Output range」ダイアログが表示されるので、
   「Start Time」と「Final Time」に直接時刻を入力して「OK」ボタンをクリックします。

    または ListViewの希望する行をクリックすると「Start Time」の値が更新されます。
   「View Final data」ボタンをクリックしてからListViewの希望する行をクリックすると「Final Time」
    の値が更新されます。

   「Prev」,「Next」ボタンをクリックすると前後にページ移動が出来ます。

    データが分割される場合は先に「Block Selector」ダイアログが開きます。
    チェックボックスを Off にするとそのブロックは出力されません。
    目的の行をクリックすると「Output range」ダイアログが開くので、そのブロックの出力範囲を設定
    できます。
      Reset ボタン：   全てのブロックをチェックし、全てのポイントを出力します。
      All Off ボタン： 全てのブロックのチェックをOffにします。

    注）NMEA入力でRMCセンテンスが無い場合は日付情報が無いので、UTCで23:59:59.9を超える部分は
        指定できません。

  8)Output File type
     KML:  Keyhole Markup Language形式で出力します。
     KMZ:  KMLをZIP圧縮して拡張子名 .kmz で出力します。
     GPX:  GPS eXchange Format形式で出力します。
     NMEA: NMEA形式のファイルを出力します。
     CSV:  CSV形式のファイルを出力します（CSVファイルからの入力は不可）。

  9)Track & Placemark
    KMLまたはKMZ形式の出力に対してのみ有効です。
    Random color:   軌跡の線色をランダムに決定します。
    Without indent: KML,KMZ,GPXファイルの先頭のタブを出力しない。
    Visible only:   チェックすると非表示の Placemarkを出力しない。
    3D: 軌跡に高度を適用します。
        チェックしない場合は地表面に表示します。
      チェックすると設定ダイアログが開きます。
      View point.
        Tilt:    視点の角度（0:真上、90:水平）。
        Clump to ground:
            Track:      軌跡を地表に固定する。
            Placemark:  プレースマークを地表に固定する。
        Range:
            Auto setting チェック時
               視点までの距離を自動設定します（X で倍率を指定できます, 初期値は 1.2）。
            Auto setting がOffの場合
               視点までの距離（m）を指定します。
        Heading:
           Auto setting チェック時
               視点の方位を出来るだけ軌跡と直角になるように自動設定します。
               North: -90..0..90度の範囲に設定します。
               East:  0..90..180の範囲に設定します。
               South; 90..+-180..-90の範囲に設定します。
               West:  -180..-90..0の範囲に設定します。
           Auto setting がOffの場合
               視点の方位を数値で指定します（0:北向き。-180..180 または 0..360）。
        Position:
           Auto setting チェック時
                   視点位置を自動設定します。
                   Start:   開始地点。
                   Halfway: 中間地点。
                   Final:   終了地点。
           Auto setting チェックがOffの場合
                   緯度、経度、高度を数値で入力します。
                   東経135度30分30秒の場合
                     135°30'30"
                     135 30 30 E
                     135.508333
                   のように入力できます。
                   入力した値は１０進数に変換されます。
      Extrude.
        Track extrude:  軌跡に地上からフェンス状に押し出します。
        Point extrude:  Placemarkに地上からの引き出し線を付加します。
      Altitude mode.
        Clamp to ground:    軌跡を地表に固定します。
        Relative to ground: 軌跡を地表からの相対地で表示します。
                            注）高度データがマイナスの部分は軌跡が表示されません。
        Absolute:           軌跡を海抜高度で表示します。
                            注）高度データが地表よりも低い部分は軌跡が表示されません。
                                グライダーまたは航空機などのデータに使用してください。
      Altitude is replaced speed.
        Replace:      チェックすると高度を速度で置き換えます。
        Coefficient:  高度への変換係数 m/(km/h)
        Offset:       高度に加算する値 m
                        Altitude = Spped * Multiplying factor + Offset

      要約表示
        T: Tilt.
        R: Range(Auto or m).
        H: Heading(N,E,S,W or degree).
        AP: Auto position, MP: Manual position.
           (St): Start point.
           (Hw): Halfway point.
           (Fin): Final point.
        TE: Track extrude.
        PE: Point extrude.
        CG: Clamp to ground, RG: Relative to ground, Abs: Absolute.
        Rp: Altitude is replaced speed.

      参照: http://homepage2.nifty.com/k8/gps/tracksample.htm

    Track:    軌跡を出力します (Line string)。
    Point:    ポイントを出力します (Place mark)。
     Balloon:   Placemarkをクリックしたときに Placemark noteを情報バルーンで表示する。
                有効にするとPlacemark noteのPoint name選択用ダイアログを開きます。
     TimeStamp: Placemark に TimeStamp要素を出力します。
                写真のジオタギングなどに必要です。
    Line width: 軌跡の線幅を指定します（0〜4を推奨）。
                PCによっては無視されることがあります。
    Alpha:    軌跡の不透明度を指定します（0〜255）。
    Color:    クリックして軌跡の線色を指定します。
    Speed:    ３種類の境界速度を指定します（左から右に昇順で入力）。
              速度単位の部分をクリックすると Km/h -> mph -> kt の順に変更できます。
    Logo:     クリックするとExternal Icon selecterフォームが開きます。
              外部アイコンとロゴマークを設定できます。
                Xpos: 0=左端、1=右端 (0〜1)。
                Ypos: 0=下端、1=上端 (0〜1)。
                Scale: 画面比(0〜1)。 -1=原寸表示。
              注）イメージファイル名には半角英数字のみでPNG形式のみ使用できます。
                  ファイル名がブランクの場合はロゴを出力しません（小文字で"Logo"を表示する）。
    Visible:  チェックしておくとデフォルトでポイントが表示されます。
    Int. Icon / Ext. Icon:
              クリックするとアイコンソースを内部アイコンと外部アイコンに切り換えます。
              個々のアイコンイメージをクリックしてアイコンを変更できます。
    POI1, 2.  チェックしておくとデフォルトでInterestポイントが表示されます
              設定ファイルで表示項目を指定できます。
    Start:    チェックしておくと最初のポイント指定したアイコンで表示します。
    End:      チェックしておくと最後のポイント指定したアイコンで表示します。
    Scale:    各アイコンの表示サイズを係数で指定します（デフォルトは 1.0）。

 10)Datetime format
    Point noteの年月日と時刻の書式を指定します。
   「Auto」をチェックすると起動時にOSから自動取得します。

    秒未満を表示するには書式文字「z」を使用します。
        例）hh:mm:ss.zzz    秒未満は３桁まで指定可能です。
    末尾が「.」のみの場合は秒未満のゼロサプレス表示を行います。
        例）h:mm:ss. の場合
               9:12:34.000 -> 9:12:34
               9:12:34:030 -> 9:12:34.03
               9:12:34:200 -> 9:12:24.2

 11)Localtime-UTC
    UTCとローカルタイムの時差を指定します（日本では 09:00:00）。
   「Auto」をチェックすると起動時にOSから自動取得します。

 12)Geoid 
    ジオイド補正方式を設定します。
    チェックボックスをチェックすると設定ダイアログがポップアップします。
    「OK」ボタンをクリックすると設定を適用します。

    ジオイドモデル
      Fixed value:       位置に関係なく固定値で補正します。
      EGM2008 2.5'x2.5': NGAの2.5'x2.5'グリッドデータを用いて補正します。
      EGM2008 1'x1':     NGAの1'x1'グリッドデータを用いて補正します。
      GSIGEO 2000:       国土地理院の1'x1.5'グリッドデータを用いて補正します。
                         初回選択時はバイナリーデータに変換するため時間がかかります。
      Custom Geoid:      CustomGeoidユーティリティーで切り出したジオイドファイルを用いて補正します。

    処理方法
      ジオイドを含むNMEAとCSV入力の場合：
        出力高度     ← GPS高度 + GPSジオイド - 計算ジオイド値
        出力ジオイド ← 計算ジオイド値
      入力にジオイドが無い場合:
        出力高度  ←  GPS高度 - 計算ジオイド値

    ジオイドチェック
      「Check」ボタンをクリックしてジオイド高の確認ができます。
      緯度経度は下記の書式で入力できます。
        37 48 51.28 S,  -37 48 51.28,  -37.814244444,  37°48'51.28"S,  -37°48'51.28"

    注1) 海抜高度で出力されたファイルに適用すると高度が不正になります(ジオイド入力なしの場合)。
    注2) 入力ファイルの拡張子名が .tks .tk1 .tk2 .tk3 の場合にはジオイド補正を適用しません。

    参考）
      EGM2008 2.5'x2.5'
        National Geospatial-Intelligence Agency (NGA) のジオイド・モデル データ。
        緯度経度2.5分のグリッドデータ(149,368,328Bytes)。
        緯度: -90〜90度、経度: 0〜359度57分30秒 の全地球データを含む。
        補間はスプライン補間（両極近辺はバイリニア補間）。
      EGM2008 1'x1'
        National Geospatial-Intelligence Agency (NGA) のジオイド・モデル データ。
        緯度経度1.0分のグリッドデータ(1,382,324,722Bytes)。
        緯度: -90〜90度、経度: 0〜359度59分 の全地球データを含む。
        補間はスプライン補間（両極近辺はバイリニア補間）。
        EGM2008 2.5'x2.5'との差は約3.5mm以内なので通常は2.5'x2.5'で十分です。
      GSIGEO 2000
        国土地理院のジオイド・モデル データ。
        緯度1分、経度1.5分のグリッドデータ(19,670,776Bytes)。
        緯度: 20〜50度、経度: 120〜150度 内の北海道、本州、四国、九州とその近海、および一部の離島を含む。
        補間はバイリニア補間（オプションでスプライン補間も可能）。
        EGM2008との差
          差異の平均値 約0.266m。
          差異の最大値 約2.7m（多良間島 北西海上）。
          北海道、本州、四国、九州の陸上での最大差 約-0.61m（納沙布岬）。
          離島陸上での最大差 約1.51m（沖永良部島）。
          東京駅での差は約0.216m。

 13) Ground level
     地表高を指定します。
    「Apply」をチェックするとデータソースの高度から地表高を減算して Placemark note
     の高度として表示します。
     注）地表高の変化には対応できないので広域データには適しません。

 14)Placemark note (KML/KMZ)
    KMLまたはKMZ形式の出力に対してのみ有効です。
    地点データ注釈の出力項目の有無を指定します。
      No.   : シーケンス番号を出力します。
      Date  : 年月日を出力します。
      Time  : 時分秒を出力します。
      Long. : 経度を出力します。
      Lat.  : 緯度を出力します。
      Speed : 速度を出力します(単位は Track - Speed オプションで選択)。
      Alt.  : 海抜高度を出力します。
      Hdop  : HDOPを出力します。
      Nsat  : 使用衛星数をを出力します。
      Temp    : 気温を出力します（WSG-1000のみ）。
                ログ出力範囲: -10〜52℃（ 2℃ステップ）。
      Pressure: 気圧を出力します（WSG-1000のみ）。
                ログ出力範囲: 589〜1100hPa（ 1hPaステップ）。
      Pressure Alt: 気圧高度を出力します（WSG-1000 NMEAのみ）。
      Voice : 音声ファイル名を出力します。

 15)CSV option (CSV)
    CSV形式の出力に対してのみ有効です。
      Index    : データ番号を出力します。
      Date+Time:「TIME」に日付と時刻を出力します。
      S/N,E/W  : 緯度経度の正負を別項目に出力します。
      Geoid    : ジオイド値を出力します。   

4.コマンド・ラインオプション
  複数のNMEAファイルおよび設定ファイル NME2KMZ.ini を指定して、複数のファイルを一括変換
  できます。
  コマンドラインに /Q を含めると自動終了します。
    例）NMEA2KMZ 06-08-01.nmea 06-08-10.nmea /Q

  パスまたはファイル名に空白を含む場合は二重引用符「"」で囲む必要があります。
    例）NMEA2KMZ "C:\Documents and Settings\User\06-08-01.nmea"

5.ドラッグ＆ドロップ
  複数の 入力ファイル（および設定ファイル NME2KMZ.ini）を実行ファイルアイコンまたは
  フォームにドラッグすることにより複数のファイルを一括変換できます。

6.設定ファイル
  終了時のオプション設定を実行ファイルと同一フォルダの NMEA2KMZ.ini に保存します。
  複数の設定ファイルを保存しておきコマンドラインまたはドラッグで選択することが出来ます。
  設定ファイルの有るフォルダが書込み禁止の場合はフォルダ %APPDATA%\NMEA2KMZ\ にコピー
  して使用します。
  %APPDATA% はDOSプロンプトで echo %APPDATA% と入力すると表示されるフォルダです。

 [Option]セクション
   Wintec WBT-200, WBT-201, RoyalTek RBT-3000, i-Blueシリーズ, HOLUX M-241 の
   ログファイル・フィルタが指定できます。
   デフォルトの設定：
     WBTmask=*.tks
     RBTmask=*.txt
     iBluemask=*.csv
     Holuxmask=*.trl
   例）
     WBTmask=read_at_*.tks

   Accel_g : 速度計算時の制限加速度を指定できます( 1 = 9.8m/S^2 )。

   WPL_Prefix: POI1 に対するNMEAのWPLセンテンスのポイント名プリフィックスを指定できます。
   WPV_Prefix: POI2 に対するNMEAのWPLセンテンスのポイント名プリフィックスを指定できます。

 [CSVtag]セクション
   Interest:  POI1 に対応する RCR(TAG)項目の文字を指定できます（デフォルト: "B")。
   Interest2: POI2 に対応する RCR(TAG)項目の文字を指定できます（デフォルト: "V")。

 [CSVitem]セクション
   CSV入力ファイルの項目名を指定できます。
   １つの項目にコンマ（,)で区切って複数の名前を指定できます。
     RCR      : 記録要因(Time, Button, Voice)。
     Date     : 日付。
     Time     : 時刻。
     Valid    : 受信品位。
     Latitude : 緯度。
     NS       : 南北記号。
     Longitude: 経度。
     EW       : 東西記号。
     Height   : 高度。
     Speed    : 速度。
     HDOP     : HDOP。
     Nsat     : 衛星数 (使用/可視)。
     Temp     : 気温。
     Barom    : 気圧。
     PAlt     : 気圧高度。
     Voice    : 音声ファイル名。
     Geoid    : ジオイド高。

 [Separator]セクション
   出力ファイル名の日付時刻の分離記号を指定できます。
     DateSep: 年月日の区切り記号（デフォルトは "-"）。
     TimeSep: 時分秒の区切り記号（デフォルトは "-"）。
     DateTimeSep: 日付と時刻の区切り記号（デフォルトは "_"）。
     StartEndSep: 開始時刻と終了時刻の間の区切り記号。
                  ブランクの場合は開始時刻のみとなります。
     空白を表す場合は「\s」を用います。

     例）DateSep=
         TimeSep=
         DateTimeSep=-
         StartEndSep=\sto\s
         の場合は「20080701-010200 to 20080701-010300」の様になります。

     注）数字と一部の記号は使用できません（ \/,;:*?"<>| 0123456789 ）。

 [Font] セクション
   表示フォントを指定します。

   日本語ロケールでのデフォルト値
     Charset=128
     FontName=MS PGothic
     FontSize=9
     DegChar=°

   その他のロケールでの初期値
     Charset=0
     FontName=Arial
     FontSize=8
     DegChar=ｰ
 
   DegChar: 度記号
   フォントやサイズによっては表示領域からはみ出すので注意してください。

 [Icon] セクション
   InternalImage= を追加すると内部アイコン選択用ビットマップファイル(.BMP)を指定できます。
   ビットマップは 512x512ピクセルで 32x32ピクセルのアイコンを256個含みます。
   通常は追加の必要はありません。

 [Note] セクション
   Placemarkの注記の小数点以下の桁数を指定します。
     LatLonDecimal: 緯度経度の小数点以下の桁数（4〜8桁）。
     AltDecimal:    標高の小数点以下の桁数（1〜8桁）。
     SpeedDecimal:  速度の小数点以下の桁数（1〜8桁）。

 [NMEAopt] セクション
   IgnoreSpace : 「1」の時はNMEAセンテンス内のスペースを削除してから解析します。
                 「0」の場合はそのまま解析します。

 [KMLKMZopt] セクション
   IconToKMZ : 「1」の時は外部アイコンをKMZファイルに組み込みます。
               「0」の場合はKMLファイルのみとします。

 [CSVopt],[KMLKMZopt],[GPXopt],[NMEAopt] セクション
   各種データの小数点以下の桁数を指定します。
     LatLonDecimal: 緯度経度の小数点以下の桁数（4〜8桁）。
     AltDecimal:    標高の小数点以下の桁数（1〜8桁）。
     SpeedDecimal:  速度の小数点以下の桁数（1〜8桁）（[KMLKMZopt]と[GPXopt]は除く）。

 [CSV_Info] セクション
   個別処理を行うGPS名称と判別用CSV項目を記述します（ユーザーが追加可能）。
   書式： GPS名称=固有CSV項目
   例）
      Wintec=Order,Date Created,Bearing,Elevation,Distance from Start,Distance from Last
      V-900, VGPS-900=TAG,LATITUDE N/S,LONGITUDE E/W,VOX
      M-241=M-241

 [CSV_Option] セクション
   GPS名称に対する処理内容を指定します（ユーザーが追加可能）。
   GPS名称「Default」は指定外の全GPSに適用します。
   書式： GPS名称=Param1,Param2,Param3,Pram4,Param5

          Param1: 時刻。
            0: UTC。
            1: ローカルタイム。
          Param2: 高度。
            0: 不定。
            1: 海抜高度 (Geoid設定を無視する)。
          Param3: 日付更新エラー。
            0: エラー無し。
            1: エラー有り。
          Param4: 速度単位。
            0: Km/h。
            1: mph。
            2: knot。
          Param5: 高度単位。
            0: m。
            1: feet。
   例）
      Default=1,0,0
      Wintec=1,1,0
      V-900, VGPS-900=0,1,1
      M-241=0,1,0

 [Geoid] セクション
   GeoidPath:  ジオイドグリッドデータのあるパスを指定します。
               空の場合は実行ファイルと同一フォルダとみなします。
   GeoidModel: 選択中のジオイドモデル。
   Digit:      NMEA出力のジオイド小数点以下の桁数。
   PostFix:    ジオイド修正を除外するファイル名の末尾を指定します。
   GeoidValue: 固定ジオイド値（ｍ）。
   Apply:      １でジオイド補正を実行する。

 [EGM2008_25],[EGM2008_10],[GSI2000] セクション
   fname: ジオイドモデルのファイル名
   nrows: Y方向のデータ数
   ncols: X方向のデータ数
   glamn: 最南端の緯度
   glomn: 最西端の経度
   dgla:  緯度のデータステップ（度）
   dglo:  経度のデータステップ（度）

   Spline: スプライン補間を用いる（GSI2000 のみ）

 [PG-S1]セクション
   Hanwha Pocket GPS S1 (PG-S1)の気圧計に関する設定を行います。
   UseBarometer:    1で気圧と気圧高度を出力します。

   Fahrenheit:      気温を華氏で入力する場合は1。

7.入力NMEAデータについて
  GGA と RMC の２つのセンテンスを使用します。
  RMC センテンスが無い場合は年月日が出力できません。
  この場合の速度は移動距離と経過時間から算出します。
  GGA センテンスが無い場合は海抜高度を出力できません。

8.入力CSVファイルについて
  １行目に項目名、2行目以降にデータを含む形式です(大文字と小文字は同一視します)。

  有効な項目名と内容
  rcr :         "B" ならインタレストポイント。
  date :        年月日 (UTC yyyy/mm/dd)。
  time :        時分秒 (UTC hh:mm:ss or hh:mm:ss.sss)。
  valid :       "no fix" または "0" なら非測位。
  lat,
  latitude :    緯度 (dd.dddddd)。
  n/s :         "N"または"S"（符号付き緯度なら不要）。
  lon,
  long,
  longitude :   経度 (ddd.dddddd)。
  e/w :         "E"または"W"（符号付き経度なら不要）。
  height :      高度 (m)。
  speed :       速度 (Km/h)。
  hdop :        HDOP。
  nsat (used/view),
  nsat,
  used sat :    使用衛星数。
  temp :        気温。
  barom :       気圧。
  palt:         気圧高度。
  vox:          音声ファイル名。
  geoid:        ジオイド高。

  1)上記以外の項目は無視します。
  2)最小必要項目は time, latitude, longitude。
  3)項目 Date が無い場合は time の内容が「yyyy/mm/dd hh:mm:ss」でも良い。
  4)各項目は任意の順序で良い。

CSVデータ例:
INDEX,RCR,DATE,TIME,VALID,LATITUDE,N/S,LONGITUDE,E/W,HEIGHT,SPEED,HDOP,NSAT (USED/VIEW),
1,T,2008/03/14,10:17:51,No fix,35.508158,N,139.614472,E,150.000 M,0.000 km/h,99.99,0(9),
2,B,2008/03/14,10:17:52,SPS,35.365336,N,139.530571,E,150.000 M,28.188 km/h,2.23,3(9),
3,T,2008/03/14,10:17:54,SPS,35.365038,N,139.530461,E,150.000 M,34.831 km/h,2.23,3(9),

9.トラブルシューティング

  1)No data のメッセージが表示されてファイルが作成されない。
    a)NMEAデータが測位状態で無い可能性が有ります。
      チェックボックス「Use Fix-data only」のチェックを外してみてください。
    b)Maskが有効でHDOPが1.0以下か、Satelites usedが12以上になっている。

  2)Can't create: XXXX のメッセージが表示されてファイルが作成できない。
    Windows vista では Program files 下へのファイル作成が出来ません。
    入力ファイルを別のフォルダにコピーしてから実行してください。

  3)トラック・アイコンが表示されない。
    KML形式で外部アイコンを使用する場合は、同一フォルダにアイコンファイルが必要です。
    KMZ形式はアイコンファイルを含むので単一ファイルで配布できます
    （但し、設定ファイルの[KMLKMZopt] セクションで IconToKMZ=0 の場合は除外）。

  4)3Dモードで軌跡が表示されない。
    GPSの高度データが地表よりも低い場合は軌跡が表示されません。
    徒歩などの地表面ログデータは3DモードをOffにしてください。

  5)ListViewのクリックで警告音が鳴る。
    下記のページを参照してください。
      MSKnowledgeBase
      http://support.microsoft.com/kb/944150/en-us/

  6)軌跡やプレースマークが３Ｄ表示されない。
    ３Ｄ設定ダイアログの設定を確認してください。
    「View point」の「Track Clamp to ground」「Placemark Clamp to ground」がチェック
     されてないか確認してください.
    「Altitude mode」で「Clamp to ground」がチェックされてないか確認してください。

  7)３Ｄモードの高さ表示が不正。
    ３Ｄ設定ダイアログの「Altitude is replaced at speed」で「Replace」がチェックされて
    ないか確認してください。

  8)バイナリーファイル gsigeome.ver4.dat が作成できない。
    Windows Vista以降で "Program files" 下にデータファイルを置いた場合は、初回のみ
    実行アイコンを右クリックして管理者権限で起動してください。

  9)プレースマーク・アイコンが表示されない。
    TimeStamp を出力した場合は Google Earth の日付時刻のスライダーで開始日時と終了日時を
    設定する必要があります。

10.ワンポイント・アドバイス

 1) 複数のログファイルを結合してKML/KMZに出力する場合の手順。
    コマンドプロンプトのDOSコマンドでファイルを結合します。

    コマンドプロンプトの起動と終了。
      起動：  スタート→すべてのプログラム→アクセサリー→コマンドプロンプト
              または スタート→ファイル名を指定して実行 で command[Enter] を入力。
      カレントドライブの移動：   D:[Enter]   で D: ドライブに移動します。
      フォルダの移動：           CD \temp    で \temp フォルダに移動します。
      コマンドプロンプトの終了： exit[Enter] で終了します。

     a) M-241の .trl ログファイルの場合（WBT-202 の .TES ファイルも同様）。
        ヘッダー無しのバイナリーファイルなのでコマンドプロンプトのCOPYコマンドで結合できます。
        例）a.trl, b.trl, c.trl を結合して t.trl を作成する場合。
            copy /B a.trl+b.trl+c.trl t.trl[Enter]

     b) M-241以外のログファイルの場合。
        NMEA2KMZで一旦 .csv または .nmea に変換してからコマンドプロンプトのCOPYコマンドで結合します。
        例）a.csv と b.csv を結合して t.csv を作成する場合。
           copy a.csv+b.csv t.csv[Enter]

     結合したログファイルから NMEA2KMZ で KML/KMZ を作成します。
     このとき「Option」の「Split Time」を充分大きくして意に反した分割を防止してください。

 2) 外部アイコンファイルについて。
    サイズが 32x32ピクセルの PNG形式のファイルを使用します。
    通常はこの中に16x16ピクセル程度のイメージを中央、上下左右のいずれかに配置します。
    BMP、Jpeg形式のファイルも可能ですが透過表示されません。
    LogoイメージにはPNG形式のみ使用できます。
    Adobe PhotoShopで作成する場合は上位レイヤーに描画し、背景レイヤーを非表示にしてからPNG形式で保存します。
    透過色はイメージの左下端のピクセルになります。
    注）イメージファイル名には半角英数字のみが使用できます。
